# AI SaaS Platform — Onaylanan Mimari Kararlar

## Mimari Yaklaşım

- **Hibrit** (ağırlıklı merkezi)
- Thin Gateway First stratejisi: Önce minimal Gateway, sonra app'ler

## Organizasyon / Ekip Yapısı

- Implicit organization pattern
- Aşama 1: Her kullanıcı = kendi bireysel organizasyonu (otomatik oluşur, kullanıcı görmez)
- Aşama 4: Ekip/organizasyon özelliği açılır (kullanıcı davet, çoklu üye)
- DB'de Organization tablosu Aşama 1'den itibaren oluşturulacak (boş kalsa bile)

## Fiyatlandırma Planları

- Başlangıç: Free + Pro
- Starter / Business / Enterprise → kullanıcı talebi geldiğinde eklenir

## Domain Stratejisi

- Hibrit subdomain yaklaşımı
- `domain.com` → Marketing / landing page
- `app.domain.com` → Platform hub (login, dashboard, billing, admin panel)
- `classifier.domain.com` → App 1 (bağımsız subdomain)
- `[appname].domain.com` → App 2, 3, 4... (her app bağımsız subdomain)
- `api.domain.com` → Gateway API (tüm app'lere hizmet)
- Auth cookie: `.domain.com` parent domain üzerinde → tek login, tüm subdomain'lere erişim
- Bundle sunulacak app'ler: aynı subdomain altında path-based grupla

## Hosting Stratejisi

- **Aşama 1-3:** Railway (hız, sıfır DevOps, managed platform)
- **Aşama 4-5:** Hetzner VPS + Dokploy'a taşınma (maliyet ↓, GDPR, tam kontrol)
- Taşınma kolay: Aynı Docker container'lar her iki ortamda çalışır

## Teknoloji Yığını (Tech Stack)

| Katman                  | Teknoloji                                                     |
| ----------------------- | ------------------------------------------------------------- |
| Gateway / Core API      | .NET 10 (LTS)                                                 |
| App Backend'leri        | Python/FastAPI (ai-classifier), .NET veya Node.js (diğerleri) |
| Frontend (Her app)      | Angular (ayrı projeler, subdomain bazlı deploy)               |
| Platform Hub UI         | Angular (login, dashboard, billing, admin)                    |
| Veritabanı              | PostgreSQL (Neon free → sonra Hetzner self-hosted)            |
| Cache / Session         | Redis (Upstash free → sonra Hetzner self-hosted)              |
| Ödeme                   | Stripe (Checkout, Customer Portal, Webhooks, Metered Billing) |
| E-posta                 | Resend veya SendGrid                                          |
| CI/CD                   | GitHub Actions                                                |
| Reverse Proxy (Hetzner) | Dokploy veya Nginx + Caddy                                    |
| Monitoring (Hetzner)    | Uptime Kuma, Grafana                                          |
