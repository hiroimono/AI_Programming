# AI SaaS Platform — Approved Architecture Decisions

## Architecture Approach
- **Hybrid** (leaning centralized)
- Thin Gateway First strategy: Build minimal Gateway first, then apps

## Organization / Team Structure
- Implicit organization pattern
- Phase 1: Each user = their own individual organization (auto-created, invisible to user)
- Phase 4: Team/organization feature enabled (invite users, multi-member)
- Organization table created from Phase 1 onward (even if empty)

## Pricing Plans
- Initial: Free + Pro
- Starter / Business / Enterprise → added when user demand arises

## Domain Strategy
- Hybrid subdomain approach
- `domain.com` → Marketing / landing page
- `app.domain.com` → Platform hub (login, dashboard, billing, admin panel)
- `classifier.domain.com` → App 1 (independent subdomain)
- `[appname].domain.com` → App 2, 3, 4... (each app independent subdomain)
- `api.domain.com` → Gateway API (serves all apps)
- Auth cookie: set on `.domain.com` parent domain → single login, access all subdomains
- Bundled apps: group under same subdomain with path-based routing

## Hosting Strategy
- **Phase 1-3:** Railway (speed, zero DevOps, managed platform)
- **Phase 4-5:** Migrate to Hetzner VPS + Dokploy (cost ↓, GDPR, full control)
- Migration easy: Same Docker containers work in both environments

## Tech Stack
| Layer | Technology |
|---|---|
| Gateway / Core API | .NET 10 (LTS) |
| App Backends | Python/FastAPI (ai-classifier), .NET or Node.js (others) |
| Frontend (Each app) | Angular (separate projects, subdomain-based deploy) |
| Platform Hub UI | Angular (login, dashboard, billing, admin) |
| Database | PostgreSQL (Neon free → later Hetzner self-hosted) |
| Cache / Session | Redis (Upstash free → later Hetzner self-hosted) |
| Payments | Stripe (Checkout, Customer Portal, Webhooks, Metered Billing) |
| Email | Resend or SendGrid |
| CI/CD | GitHub Actions |
| Reverse Proxy (Hetzner) | Dokploy or Nginx + Caddy |
| Monitoring (Hetzner) | Uptime Kuma, Grafana |

## Roadmap Summary

| Phase | Duration | Content | Hosting | Cost |
|---|---|---|---|---|
| 1 | ~2 weeks | Gateway MVP (Auth, JWT, OAuth, RBAC) + App 2 kickoff | Railway | ~$15-20/mo |
| 2 | ~3-4 weeks | User Management, Admin Panel, Email, Domain setup, App 2 completion | Railway | ~$15-20/mo |
| 3 | ~4-6 weeks | Stripe Billing, Plans, Usage limits, App 3 | Railway | ~$40-60/mo |
| 4 | ~4-6 weeks | Hetzner migration, Multi-app consolidation, Org/Team, Apps 4-5 | Hetzner | ~€15-25/mo |
| 5 | Ongoing | MFA, Audit, Analytics, M2M API, Advanced admin, App 6 | Hetzner | ~€15-25/mo |

## M2M / API Exposure
- Implemented in Phase 5
- API Key management, rate limiting, webhooks, OpenAPI documentation
